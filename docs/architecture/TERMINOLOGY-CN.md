# 产品术语标准

英文主文档：[TERMINOLOGY.md](TERMINOLOGY.md)

## 标准产品定位

Eurogas Nexus 是欧洲天然气市场情报、优化和决策支持平台。

产品支持市场分析、基础设施可视化、路径经济性、合同和资源评估、优化以及由交易员复核的决策。它不是交易场所、订单路由器、提名系统、结算平台或 ETRM 替代品。

## 必须采用的术语

| 避免使用 | 改用 |
|---|---|
| 仅供研究 / research only | 决策支持 / 必须人工复核 |
| 研究型平台 | 决策支持平台 |
| 研究工具 | 决策支持工具 |
| 研究工作区 | 分析工作区或优化工作区 |
| 研究助手 | 市场分析助手或决策助手 |
| 自动交易建议 | 由交易员复核的决策支持输出 |

## 代码和 API 规则

- 新请求体、领域模型、存储记录和业务数据载荷不得新增 `research_only` 字段。
- 输出必须由交易员或运营人员复核时，使用 `human_review_required`。
- 存储记录或未来 API 合同需要明确治理分类时，使用 `decision_support_only`。
- 公共 API 信封暂时保留 `meta.research_only`，用于兼容现有 Web `ApiMeta` 和仓库合同；不得把该字段复制到端点的业务数据对象。
- 移除兼容字段必须通过单独迁移，同时更新后端信封、Web 类型和使用方、仓库规则、合同测试以及兼容文档。
- 不得暗示优化结果会执行交易、提交提名、预订管容或形成正式审批。

## 优化能力命名

优先使用：路径优化、资源池优化、管容优化、合同优化、储气调度、情景分析和风险优化。

为了兼容已有技术路径和模块名，可以保留 `Route Cost`；新的用户界面和文档应优先使用“路径优化”或“路径经济性”。

## 人工复核边界

所有优化、情景、分析、路径、组合和报告输出都必须经过人工复核。该要求应通过结果元数据和产品文档表达，不得为此向新的业务数据对象添加 research-only 字段。
