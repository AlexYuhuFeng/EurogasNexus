# 地图优先交易员驾驶舱规格 - CN

## 目的

Eurogas Nexus 首页是地图优先的天然气交易决策驾驶舱，不是普通仪表盘。它的
目的，是让交易员在一个视觉主界面中看到上游资源组合、路线可用性、实时市场
变化、策略运行状态、预警和指示性 PnL。

## 首页布局

首屏必须以欧洲天然气地图为主。地图必须展示：

- 上游资源合同和资源池敞口；
- LNG 终端、海滩交付点、枢纽、互联点、储气库和主要路线走廊；
- 从资源起点到销售/交付目标的路线选项；
- 可用路线和被阻断路线；
- TSO 准入约束；
- 后端有数据时的容量和停运状态；
- 与所选组合路线相关的实时市场变化；
- 策略状态和预警覆盖层。

地图不能只是装饰背景，必须是主要工作界面。

## 地图上方价格与进程条

地图上方放置紧凑实时条，显示：

- 组合敞口数量；
- 相关 hub/day-ahead 参考价格；
- 有授权时的 ICE OCM 或其他日内报价；
- 涉及跨币种经济性时的 ECB FX；
- 实时指示性 PnL；
- 当前策略运行状态；
- 最新决策支持信号；
- 预警数量。

后端提供来源和新鲜度时，每个数值都必须显示来源和新鲜度。

## 地图交互

必须支持：

- 搜索终端、hub、TSO、平衡点、气田或数据源；
- 切换管网、LNG、互联点、hub、储气、天气、合同、市场、路线和预警图层；
- 点击路线或资产打开检查面板；
- 视觉化比较路线备选方案；
- 高亮被阻断原因，例如缺少 TSO 准入、缺少费率、缺少容量、缺少终端窗口、
  缺少供应商凭证或数据过期；
- 只通过后端 API 调用路线成本、LNG regas readiness、资源池或策略评估。

## 独立详情标签或窗口

详细工作不能挤占首页地图。必须提供独立标签或窗口：

- 价格监控；
- 组合/资源池组合；
- 路线成本和 LNG regas 分析；
- 策略回测、影子运行和实时监控；
- 导入的外部订单记录；
- 预警中心；
- 来源血缘和授权状态；
- 术语表；
- 用户手册；
- 凭证和设置。

V1 中订单记录仅为导入/参考记录。产品不得下单、路由订单、修改订单或取消订单。

## 策略 UX

策略界面必须支持：

- 回测模式；
- 影子运行模式；
- 实时监控模式；
- 已配置资源池；
- 例如 15:00-17:00 的交易窗口；
- 例如 5 分钟的 bar 周期；
- SAP 与 ICIS 日前价格对比 ICE OCM、均值回归、评分、最佳时间桶和加权组合；
- 最大 OCM 分配、最低日前分配、单一市场最大数量、最低预期利润、止损、
  数据过期阻断和 TSO 准入阻断等风控；
- 带依据、缺失输入、预警、来源和人工复核状态的纸面分配目标。

UI 必须使用决策支持语言，例如信号、候选、选项、目标、复核。不得使用执行性语言。

## 视觉质量标准

商业交付质量要求：

- 专业交易工作站式布局，信息密度高但可读；
- 克制的配色和清晰的语义状态；
- 不做营销首页、landing page 或装饰性填充；
- 稳定的响应式尺寸；
- 支持 light/dark/system 主题；
- 所有可见文字通过 i18n 支持英文和中文；
- 地图和数据面板在桌面与笔记本屏幕上可用；
- 移动端/平板端降级为地图优先标签页，而不是压缩仪表盘。

## 数据边界

驾驶舱只能消费后端 `/api/v1` 或 SDK 表面。不得直接读取 PostgreSQL、后端文件、
供应商文件、`.env` 或凭证。
