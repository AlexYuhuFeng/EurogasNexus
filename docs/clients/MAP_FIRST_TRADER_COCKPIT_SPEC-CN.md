# 地图优先交易员驾驶舱规格 - CN

## 目的

Eurogas Nexus 首页是地图优先的交易员决策驾驶舱，不是普通仪表盘。它的目的，是让天然气交易员在一个视觉主屏上理解上游资源池、路线可用性、实时市场变化、策略运行状态、预警和指示性 PnL。

首页必须是资源池原生设计。所有有效买入合同先被理解为组合资源池中的可用资源；地图和右侧决策栏展示这组资源应如何销售，而不是只展示“当前选中的一份合同”。

## 首页布局

首屏必须以欧洲天然气地图为主。地图必须展示：

- 上游买入合同形成的资源池敞口；
- LNG 接收站、beach 交付点、hub、互联点、储气库和平衡区；
- 从资源位置到销售市场的候选路径；
- 可用路线、被阻断路线和分拆销售路径；
- TSO 准入约束；
- 后端有数据时的管容、流量、停运和费率状态；
- 与资源池相关的实时价格、价差和策略状态；
- 预警覆盖层。

资源标记必须指出资源池可用的位置，例如 beach 交付点、LNG 接收站、hub、互联点、客户录入的交付点或 title-transfer 点。推荐销售路径必须同时可见，使交易员能看到整体分配、分拆路线和被阻断方案。

地图不能只是装饰背景，必须是主要工作界面。

## 地图上方价格和过程条

地图上方放置紧凑实时条，显示：

- 资源池总量和未分配数量；
- 相关 hub/day-ahead 参考价格；
- 有授权时的 ICE OCM 或其他日内标记；
- 涉及跨币种经济性时的 ECB FX；
- 实时指示性 PnL；
- 导入的外部屏幕订单姿态和组合 PnL 快照；
- 当前策略运行状态；
- 最新决策支持信号；
- 预警数量。

后端提供来源和新鲜度时，每个数值都必须显示来源和新鲜度。

## 地图交互

必须支持：

- 搜索终端、hub、TSO、平衡点、气田或数据源；
- 切换管网、LNG、互联点、hub、储气、天气、合同、市场、路线和预警图层；
- 点击路线或资产打开检查面板；
- 视觉化比较多条路线；
- 高亮阻断原因，例如缺少 TSO 准入、缺少费率、缺少管容、缺少接收站窗口、缺少供应商凭证或数据过期；
- 只通过后端 API 启动路线成本、LNG regas readiness、资源池优化或策略评估。

存在订单/PnL 上下文时，地图必须在相关节点之间画出高亮路线，并显示简洁标签和 PnL 数值。该动画仅用于决策支持和视觉理解，不代表提名、调度、订单路由或可执行交易路径。动画必须尊重 `prefers-reduced-motion`。

## 资源池决策栏

首页右侧栏必须总结组合决策：

- 资源池总预期 PnL；
- 已分配数量和未分配数量；
- 每条推荐销售路径的目标市场、分配数量、路线成本、销售价格、边际 netback、PnL 贡献、容量限制、TSO 准入状态和预警；
- 被阻断路线及明确阻断原因；
- 合同层 PnL 归因作为下钻信息，而不是首要决策。

如果最便宜路线容量有限，UI 必须显示该路线的部分分配，并显示剩余数量的下一最佳处理方式。若剩余数量绕路销售不经济，建议应显示本地销售、其他市场销售或保留/不销售，而不是强制走亏损路线。

## 合同录入

合同细节应放在独立 Contracts 页面。页面必须遵循 `docs/contracts/21_RESOURCE_POOL_CONTRACT-CN.md` 中的 EFET 风格结构：Agreement、Product and term、Delivery、Quantity/tolerance、Price、Costs、Capacity rights、Settlement/cash 和 Restrictions。首页只消费已存入 PostgreSQL 并通过 `/api` 暴露的资源池状态。

## 独立详情页面

详细工作不能挤占首页地图。必须提供独立页面：

- 价格监控；
- 组合和资源池；
- 合同录入；
- 路线成本和 LNG regas 分析；
- 策略回测、影子运行和实时监控；
- 导入的外部订单记录；
- 预警中心；
- 数据源血缘和授权状态；
- 术语库；
- 用户手册；
- 凭证和设置。

V1 中订单记录仅为导入/参考记录。产品不得下单、路由订单、修改订单或取消订单。

## 策略 UX

策略页面必须支持回测、影子运行、实时监控、配置的资源池、交易窗口、bar 周期、SAP/ICIS/ICE OCM 对比、均值回归、评分、最佳时段、加权组合和风控限制。UI 必须使用决策支持语言，例如信号、候选、选项、目标、复核；不得使用执行性语言。

## 视觉质量标准

商业交付质量要求：

- 专业交易工作站式布局，信息密度高但可读；
- 克制配色和清晰语义状态；
- 不做营销首页、landing page 或装饰性填充；
- 稳定的响应式尺寸；
- 支持 light/dark/system 主题；
- 所有可见文字通过 i18n 支持英文和中文；
- 地图和数据面板在桌面与笔记本屏幕上可用；
- 移动端和平板降级为地图优先标签页，而不是压缩仪表盘。

## 数据边界

驾驶舱只能消费后端 `/api` 或 SDK 表面。不得直接读取 PostgreSQL、后端文件、供应商文件、`.env` 或凭证。
